seed store — a real-P2P storage node in your browser

Each tab is a full storage node. A file you drop here is encrypted and erasure-coded across the cohort's nodes; any tab can reconstruct it. Two ways to reach the cohort: WebRTC — tabs find each other through a WebSocket relay (signaling only) and then talk directly, peer-to-peer, no server in the data path (the same RtcNetwork a Node peer joins via node-datachannel, README §13.6); or Direct WebSocket — dial a known set of natively-reachable nodes (a public IP / LAN / port-forward) straight at their --ws-listen port, no relay and no STUN (WsNetwork).

relay room

Dial natively-reachable nodes started with seedloader --ws-listen 0.0.0.0:<port>. One pubkey@host:port per line (or comma-separated). Need at least k+m distinct holders.

cohort bundle author:

Descriptor signatures are scoped to the deployment's bundle author (README §16): a holder running a signed bundle (a seedloader) only admits blocks signed under that author's scope. This is filled in for you from the deployment's manifest.bundle; override only to join a different cohort, or clear it when every holder is another browser tab / a default zero-author StorageNode. A mismatch makes holders silently decline every block.
